# Session Project Separation Todo

## 1. 메모 목적

이 문서는 specdrive와 실제 프로젝트 문서가 분리될 때 `session` 단계 구조를 다시 검토하기 위한 보류 메모다.

현재 저장소는 specdrive 자체 문서와 검증용 프로젝트 문서가 한 저장소 안에 함께 있어 세션 복구 문맥이 무거워질 수 있다.  
이 문제는 현재 즉시 해결할 작업이 아니라, 나중에 specdrive와 프로젝트를 분리하는 시점에 다시 판단한다.

---

## 2. 현재 관찰

현재 세션 복구는 다음 문맥이 함께 섞일 수 있다.

- specdrive 자체 운영 문서
- repo-local skill 문서
- 프로젝트 문서
- 프로젝트 work 문서
- 상태 복구 문서

이 구조는 통합 검증 단계에서는 유용하지만, 세션 시작 시 불필요한 문맥이 함께 들어와 무거워질 수 있다.

---

## 3. 후속 검토 방향

specdrive가 실제 프로젝트와 분리되면 `session` 은 단순 복구 명령보다 다음 흐름을 감싸는 운영 단계로 다시 볼 수 있다.

```text
doc
  -> plan
  -> dev
```

즉, 분리 이후의 session은 다음을 검토해야 한다.

- specdrive 자체 세션과 프로젝트 작업 세션을 분리할지
- 프로젝트 세션에서 `doc / plan / dev` 흐름을 어떻게 이어줄지
- 세션 복구 문서가 specdrive core 기준인지, 프로젝트 기준인지 구분할지
- 프로젝트별 active context를 어디서 관리할지
- 통합 저장소 기준의 현재 `$session start` 흐름을 분리형 구조에 맞게 바꿀지

---

## 4. 현재 결정하지 않는 것

현재는 다음을 결정하지 않는다.

- session 구조 개편
- specdrive와 프로젝트의 실제 저장소 분리 방식
- 프로젝트별 세션 상태 파일 구조
- `$session` action 변경
- 기존 session 문서 참조 변경

이 문서는 참조 흐름에 연결하지 않는다.  
프로젝트 분리 작업을 시작할 때 개발자가 명시적으로 요청하면 다시 검토한다.
