# specdrive/skills/doc/reinforce.md

## 1. skill 목적

이 skill은 `doc reinforce` 작업에서 사용하는  
**기본 문서 보강 규칙 자산**이다.

목적은 다음과 같다.

- 기존 문서 초안을 더 읽기 쉽고 더 작업 가능하게 만든다.
- 문서 목적, 범위, 구조, 누락 항목을 점검한다.
- 현재 단계에서 필요한 최소 보강안을 만든다.
- 문서를 과도하게 재설계하지 않도록 기준을 제공한다.

이 skill은 문서를 확정하지 않는다.  
이 skill의 결과는 항상 사람이 검토해야 한다.

---

## 2. 현재 1차 테스트 대상

현재 이 skill의 첫 테스트 대상 문서는 다음과 같다.

- `docs/projects/board/01-overview.md`

이 문서는 board 프로젝트의 가장 앞쪽 개요 문서이며,  
현재 `doc reinforce` 최소 흐름을 검증하기에 적절한 대상이다.

---

## 3. 입력 기준

이 skill은 기본적으로 다음 입력을 전제로 한다.

- 대상 문서 본문
- 대상 문서 경로
- 관련 상위 진입 문서
  - `README.md`
  - `AGENTS.md`
  - `docs/AI_CONTEXT.md`
  - `docs/projects/board/README.md`
  - `docs/projects/board/AGENTS.md`

현재 단계에서는 입력을 과도하게 넓히지 않는다.  
우선은 대상 문서와 핵심 진입 문서를 중심으로 보강한다.

---

## 4. 보강 목표

이 skill의 보강 목표는 다음과 같다.

- 문서 목적을 더 선명하게 만든다.
- 현재 범위와 비포함 범위를 더 분명하게 만든다.
- 문서 구조를 더 읽기 쉽게 정리한다.
- 빠진 핵심 항목이 있으면 최소 수준에서 제안한다.
- 후속 문서로 자연스럽게 이어질 수 있도록 연결 지점을 정리한다.

중요한 점은 다음과 같다.

- 새로운 기능을 과도하게 추가하지 않는다.
- 문서를 완전히 다른 문서로 바꾸지 않는다.
- 현재 단계보다 너무 앞선 상세 설계를 넣지 않는다.

---

## 5. 보강 시 확인할 것

보강할 때는 아래를 먼저 확인한다.

1. 이 문서의 목적이 한눈에 드러나는가?
2. 현재 단계의 범위와 후속 단계가 구분되는가?
3. 문장이 모호하거나 중복되지는 않는가?
4. specdrive 문서 내용이 project 문서로 섞여 들어가지는 않았는가?
5. board 프로젝트 문서로서 정체성이 유지되는가?
6. 다음 요구사항 / 설계 / 구현 계획 문서로 이어질 기준점이 되는가?

---

## 6. 보강 방식

현재 기준 보강 방식은 다음을 따른다.

- 기존 문서 구조를 먼저 존중한다.
- 필요한 경우 제목/섹션을 소폭 정리한다.
- 누락 항목은 최소 범위에서 보강한다.
- 표현은 더 명확하게 정리하되, 원래 문서의 의도를 훼손하지 않는다.
- 큰 구조 변경이 필요하면 직접 확정하지 말고 제안 수준으로 남긴다.

---

## 7. 금지 사항

이 skill은 다음을 하지 않는다.

- 문서를 confirm 된 기준 문서처럼 확정하지 않는다.
- 현재 없는 상세 요구사항을 임의로 크게 만들어 넣지 않는다.
- 아직 정해지지 않은 API / DB / 화면 설계를 확정해 넣지 않는다.
- board 문서에 specdrive 내부 운영 규칙을 과도하게 가져오지 않는다.
- 문서 전체를 새로 쓰듯이 갈아엎지 않는다.

---

## 8. 기대 출력

이 skill의 기대 출력은 다음과 같다.

- 보강된 문서 초안
- 누락되었지만 현재 단계에 필요한 최소 항목 보완
- 모호한 표현 정리
- 사람이 바로 검토할 수 있는 수준의 문서 개선안

즉 출력은 “새 문서 창작”이 아니라  
**현재 문서를 더 명확하고 더 작업 가능하게 만드는 보강안**이어야 한다.

---

## 9. 사람 검토 포인트

이 skill 실행 결과를 검토할 때는 아래를 확인한다.

- board의 정체성이 흔들리지 않았는가?
- 현재 범위가 더 분명해졌는가?
- 후속 문서 작성의 기준점 역할을 할 수 있는가?
- 과도한 확장이나 조기 상세화가 들어가지 않았는가?
- 원래 문서의 의도와 방향을 해치지 않았는가?

---

## 10. 최종 정리

`specdrive/skills/doc/reinforce.md` 는  
현재 `doc reinforce` 흐름에서 사용하는  
가장 기본적인 문서 보강 규칙 자산이다.

현재 첫 테스트는  
`docs/projects/board/01-overview.md` 를 대상으로 하며,  
목표는 완벽한 자동화보다  
**반복 가능한 최소 보강 흐름을 안정화하는 것**이다.
