# specdrive/skills/doc/confirm.md

## 1. skill 목적

이 skill은 `doc confirm-prompt` 작업에서 사용하는  
**기본 문서 확정 판단 규칙 자산**이다.

목적은 다음과 같다.

- 현재 문서를 기준 문서로 볼 수 있는지 판단을 돕는다.
- reinforce 결과와 현재 문서의 상태를 검토한다.
- confirm 전에 확인해야 할 최소 체크포인트를 제공한다.

이 skill은 문서를 자동 확정하지 않는다.  
최종 반영 판단은 항상 사람이 수행한다.

---

## 2. 현재 1차 테스트 대상

현재 이 skill의 첫 테스트 대상 문서는 다음과 같다.

- `docs/projects/board/01-overview.md`

---

## 3. 확인 목표

이 skill의 확인 목표는 다음과 같다.

- 문서 목적이 충분히 선명한가?
- 현재 범위와 비포함 범위가 구분되는가?
- project 문서로서 정체성이 유지되는가?
- 후속 요구사항 / 설계 / 구현 계획 문서의 기준점 역할을 할 수 있는가?
- 아직 reinforce 가 더 필요한 수준은 아닌가?

---

## 4. 금지 사항

이 skill은 다음을 하지 않는다.

- 사람 승인 없이 문서를 confirm 처리하지 않는다.
- 확인되지 않은 상세 설계를 임의로 확정하지 않는다.
- 문서 목적과 범위를 넘어서는 확장을 강요하지 않는다.

---

## 5. 기대 출력

이 skill의 기대 출력은 다음과 같다.

- confirm 체크리스트
- 반영 가능 / 보류 필요 판단 보조
- confirm-prompt 이후 남은 수정 포인트 요약

---

## 6. 최종 정리

`specdrive/skills/doc/confirm.md` 는  
`doc confirm-prompt` 단계에서 사람이 최종 판단을 내릴 수 있게 돕는  
기본 confirm 규칙 자산이다.
