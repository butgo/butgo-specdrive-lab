# specdrive/skills/session/start.md

## 1. skill 목적

이 skill은 `session start` 작업에서 사용하는
**세션 복구 시작 규칙 자산**이다.

목적은 다음과 같다.

- 새 세션에서 현재 저장소 상태를 빠르게 복구한다.
- 필요한 기준 문서만 직접 읽게 한다.
- 현재 focus, 다음 진입점, 주의해야 할 변경 범위를 먼저 요약하게 한다.
- 개발자 요청 전에는 파일을 직접 수정하지 않도록 경계를 고정한다.

이 skill은 작업을 자동으로 진행하거나 문서를 수정하지 않는다.

---

## 2. 기본 읽기 순서

먼저 다음 문서를 직접 읽는다.

1. `README.md`
2. `AGENTS.md`
3. `docs/AI_CONTEXT.md`
4. `docs/specdrive/AGENTS.md`
5. `docs/specdrive/session-stage.md`

작업 대상 영역이 정해져 있으면 다음 문서를 추가로 읽는다.

1. 해당 영역의 `AGENTS.md`
2. 해당 영역의 `README.md`
3. 해당 영역의 `index.md`
4. 현재 수정 또는 작성할 대상 문서

문서 본문을 사용자에게 새로 붙여넣으라고 요구하지 말고,
저장소의 파일을 직접 읽어 필요한 최소 문맥만 사용한다.

---

## 3. 복구 출력 기준

세션 복구 후 먼저 다음 세 가지를 짧게 정리한다.

1. 현재 focus
2. 다음 진입점
3. 주의해야 할 변경 범위

필요하면 현재 브랜치, Git 상태 요약, 변경 영역 요약을 보조 정보로 반영한다.
상세 변경 파일 목록은 필요할 때만 확인한다.

---

## 4. 중단 및 확인 지점

다음 경우에는 먼저 개발자에게 확인한다.

- 신규 문서 생성
- 문서 역할 변경
- 요구사항에서 설계로 넘어가는 전환
- 설계에서 구현 계획으로 넘어가는 전환
- 상태 문서나 인덱스의 활성 단계 또는 다음 진입점 변경

개발자가 명시적으로 요청하기 전에는 파일을 직접 수정하지 않는다.

---

## 5. 금지 사항

이 skill은 다음을 하지 않는다.

- `doc` 단계의 문서 보강이나 확정을 대신하지 않는다.
- `dev` 단계의 task 분해나 구현 실행을 대신하지 않는다.
- `git` 단계의 브랜치명, commit message, PR 메시지 생성을 대신하지 않는다.
- `docs/history/**` 를 기본 복구 범위로 열람하지 않는다.
- 현재 결정되지 않은 내용을 확정처럼 말하지 않는다.

---

## 6. 최종 정리

`session start` 는 작업 실행 명령이 아니라
현재 상태를 복구하고 안전한 진입점을 확인하는 운영 보조 흐름이다.

따라서 이 skill의 결과는
작업 시작 전의 짧은 상태 복구 요약이어야 한다.
