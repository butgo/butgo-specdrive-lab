# AGENTS.compact.md

## 1. 역할

이 문서는 루트 `AGENTS.md`의 대체 원본이 아니다.

이 문서는 Codex 기본 주입용 compact 규칙이다.  
긴 원본을 매번 읽지 않고도 저장소 공통 작업 기준을 빠르게 복구하기 위해 사용한다.

원본 `AGENTS.md`는 최상위 기준 문서로 그대로 유지한다.

---

## 2. 저장소 성격

butgo-specdrive는 개발 스펙 문서를 기반으로 AI 협업을 실행하는 도구다.

현재 저장소는 다음 성격을 가진다.

- 완성된 SaaS 제품 저장소가 아니다.
- core workflow / engine / repo-local Codex skill 중심 운영체계 검증 단계다.
- 실제 애플리케이션 프로젝트를 함께 두고 specdrive 운영 방식을 검증한다.
- 현재 AI 엔진 기준은 Codex다.
- 현재 실행 기준은 repo-local Codex skill 직접 사용이다.

이 저장소를 일반적인 웹 서비스나 백오피스 SaaS 프로젝트처럼 해석하지 않는다.

---

## 3. 문서 계층

문서 규칙의 기본 우선순위는 다음과 같다.

1. 루트 `AGENTS.md`
2. 영역 전용 `AGENTS.md`
3. 개별 문서
4. 작업 메모, 임시 노트, 대화 초안

compact 문서는 원본의 빠른 실행 요약본이며, 원본보다 우선하지 않는다.

---

## 4. 작업 단계 구분

작업 단계는 섞지 않는다.

- `doc`: 문서를 구현 가능한 상태로 만드는 단계
- `dev`: 확정된 문서를 기준으로 구현을 진행하는 단계
- `session`: 세션 시작/종료와 전달을 돕는 메타 운영 단계
- `git`: commit, push, PR 같은 전달 단위 준비 단계

`session save`는 `doc history-save`를 대체하지 않는다.  
`git`은 문서 확정이나 세션 저장을 대신하지 않는다.

---

## 5. 문서 우선 원칙

이 저장소의 기본 흐름은 다음이다.

1. 먼저 문서로 구조와 경계를 정한다.
2. 문서를 기준으로 AI 입력을 구성한다.
3. 결과를 코드나 후속 문서에 반영한다.
4. 중요한 판단과 변경 이력을 남긴다.

구현은 문서를 따른다.  
문서가 확정되지 않았으면 구현 단계로 성급히 넘어가지 않는다.

---

## 6. 개발자 승인 필요 지점

다음 작업은 개발자 확인 또는 명시 요청 후 진행한다.

- 신규 문서 생성
- 기존 문서 역할 변경
- 요구사항에서 설계로 전환
- 설계에서 구현 계획으로 전환
- 상태 문서의 활성 단계나 다음 진입점 변경
- history snapshot 생성
- Git commit, push, PR 생성
- 현재 범위를 넘는 확장 결정

개발자가 명시적으로 요청한 범위 안에서는 진행할 수 있다.

---

## 7. history 기본 제외 원칙

`docs/history/**`는 현재 기준 문서가 아니라 과거 판단과 산출물을 보존하는 영역이다.

기본 원칙은 다음과 같다.

- 평소 세션 복구나 문서 정비의 기본 점검 범위에 넣지 않는다.
- 현재 규칙, 현재 상태, 현재 문서 구조는 먼저 현재 기준 문서에서 확인한다.
- 개발자가 특정 이력 확인, 근거 추적, 비교 검토를 명시적으로 요청한 경우에만 읽는다.
- history는 현재 문서를 대체하지 않는다.

---

## 8. 필요한 문맥만 읽는 원칙

AI에게는 필요한 문맥만 제공한다.

기본 순서는 다음과 같다.

1. compact 규칙 문서
2. 대상 영역의 compact 규칙 문서
3. 대상 문서
4. 필요한 참조 문서

무조건 많은 문서를 읽지 않는다.  
참조 문서, bundle 문서, history 문서는 작업 요청에 포함된 경우에만 읽는다.

---

## 9. 원본 AGENTS.md를 읽어야 하는 경우

다음 경우에는 compact 문서만으로 판단하지 말고 원본 `AGENTS.md`를 읽는다.

- compact 문서와 실제 작업 판단이 충돌하거나 불분명한 경우
- 신규 작업 단계나 문서 역할을 정의해야 하는 경우
- 공통 규칙 자체를 수정하는 경우
- 영역 전용 규칙과 루트 규칙의 우선순위를 판단해야 하는 경우
- 승인 필요 여부가 애매한 경우
- 장기 운영 기준이나 금지 사항을 세밀하게 확인해야 하는 경우

---

## 10. 현재 결정

- compact 문서는 기본 주입용 요약본으로 사용한다.
- 원본 `AGENTS.md`는 삭제하거나 축약하지 않는다.
- `docs/history/**`는 기본 context에서 제외한다.
- `.agents/skills/**`는 현재 repo-local Codex skill 실행 기준 원본으로 본다.
- `specdrive/codex-skills/**`는 mirror/export 성격이면 기본 context에서 제외한다.

---

## 11. 후속 후보

- context bundle이 compact 문서를 원본보다 우선 선택하도록 자동화한다.
- thin skill + template 분리로 skill 본문 토큰을 줄인다.
- compact 문서와 원본 문서의 drift 점검 절차를 마련한다.
