# SpecDrive VSCode 플러그인 수동 테스트 가이드 (Manual Testing Guide)

이 문서는 순수 비즈니스 로직(파서, 상태 제어기)이 연동된 `specdrive-vscode` 플러그인을 로컬 개발 환경에서 직접 실행하고 테스트하는 방법을 안내합니다.

## 1. 사전 준비 (Prerequisites)
- VSCode 에디터가 열려 있어야 합니다.
- `specdrive-vscode` 폴더 내에서 `npm install` 이 정상적으로 완료되어 있어야 합니다.

## 2. 테스트 실행 단계

### 단계 1: 확장 프로그램 개발 호스트(Extension Host) 실행
1. 에디터에서 `specdrive-vscode/src/extension.ts` 파일을 엽니다.
2. 키보드의 **`F5`** 키를 누릅니다. (또는 좌측 Run & Debug 메뉴에서 실행)
3. "확장 프로그램 개발 호스트(Extension Development Host)"라는 타이틀이 붙은 **새로운 VSCode 창**이 팝업으로 뜹니다.

### 단계 2: 더미 기획서(Markdown) 준비
새로 뜬 확장 프로그램 창에서 다음 작업을 수행합니다.
1. `Ctrl+N`을 눌러 새 텍스트 파일을 만듭니다.
2. 우측 하단 언어 모드를 클릭하여 **Markdown**으로 변경하거나, `test-spec.md` 라는 이름으로 저장합니다.
3. 문서 안에 파서가 인식할 수 있도록 아래 포맷으로 텍스트를 작성합니다.

```markdown
# 프로젝트 개요
이것은 테스트 문서입니다.

## Task
간단한 로그인 폼 작성 및 상태 제어기 연동 테스트
```

### 단계 3: 파이프라인 트리거 (Command 실행)
1. 작성한 마크다운 파일이 화면에 활성화(열려 있는) 상태여야 합니다.
2. **`Ctrl+Shift+P`** (Mac: `Cmd+Shift+P`)를 눌러 명령어 팔레트를 엽니다.
3. **`SpecDrive: Run Tasks`** 를 검색하여 엔터를 칩니다.

### 단계 4: 상태 전이 및 하네스(Harness) 동작 확인
화면 우측 하단 알림창(Notification)의 변화를 관찰합니다.

1. `SpecDrive: Task 파싱을 시작합니다...` 알림이 발생합니다. (**PARSING** 상태 진입)
2. 약 0.5초(가짜 AI 응답 시간) 후, **[코드 생성 완료] 승인하시겠습니까?** 라는 알림과 함께 **[승인] / [반려]** 버튼이 나타납니다. (**REVIEWING** 상태 일시 정지)
   - *이 구간이 코드가 파일에 무단으로 적용되는 것을 막는 핵심 하네스(통제 지점)입니다.*
3. **[승인]** 버튼을 클릭하면 `코드가 적용되었습니다!` 알림이 뜨며 워크플로우가 종료됩니다. (**APPLYING** -> **IDLE** 상태 복귀)
4. **[반려]** 버튼을 누를 경우, 다시 상태를 되돌려 재생성을 시도하는 로직으로 롤백됨을 확인할 수 있습니다.

---
> **Troubleshooting**
> * 만약 `F5`를 눌렀을 때 실행되지 않는다면 하단 터미널에서 `npm run build`를 수동으로 한 번 실행해 주세요.
> * 마크다운 파일이 아닌 곳에서 실행하면 오류 메시지가 정상적으로 표시되는지(방어 로직)도 테스트해 보실 수 있습니다.
