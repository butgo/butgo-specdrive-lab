# docs/projects/board/02-requirements.md

## 1. 문서 목적

이 문서는 board 프로젝트의 **현재 단계 최소 요구사항 문서**다.

목적은 다음과 같다.

- board가 현재 단계에서 무엇을 해야 하는지 정리한다.
- 현재 포함 범위와 제외 범위를 요구사항 관점에서 구분한다.
- 후속 설계 문서가 받아야 할 입력을 정리한다.
- 구현 계획 문서가 작업 단위를 나눌 수 있는 기준점을 제공한다.

이 문서는 상세 설계 문서가 아니다.  
또한 API, DB, 패키지 구조를 확정하는 문서도 아니다.  
현재는 **무엇을 만들어야 하는가**를 최소 범위로 고정하는 데 집중한다.

---

## 2. 현재 단계 한 줄 요약

board는 문서 기반 AI 협업 흐름을 실제 애플리케이션 개발에 적용해 보기 위한  
**최소 게시판 애플리케이션 요구사항 검증 대상**이다.

---

## 3. 현재 요구사항 정리 기준

현재 요구사항은 다음 기준으로 정리한다.

- 너무 이른 상세 설계로 넘어가지 않는다.
- 실제 구현과 검증이 가능한 최소 기능만 먼저 다룬다.
- 현재 범위와 후속 후보를 섞지 않는다.
- 공통 standards는 참조하되, 이 문서에서는 board에 필요한 요구사항만 다룬다.

즉 현재 단계에서는 “완성형 게시판 제품 요구사항”보다  
**최소 검증 가능한 board 기능 범위**를 먼저 고정하는 것이 중요하다.

---

## 4. 현재 목표

현재 board가 달성하려는 목표는 다음과 같다.

- 최소 게시판 애플리케이션의 핵심 흐름을 정의한다.
- 문서 우선 개발 방식으로 요구사항에서 설계/구현 계획으로 이어지는 기준을 만든다.
- 이후 `dev` 단계에서 task 분해가 가능한 수준의 기능 범위를 고정한다.

현재 단계에서는 기능 수를 넓히기보다,  
**작은 범위라도 명확하게 정의된 요구사항 묶음**을 만드는 것이 우선이다.

---

## 5. 현재 포함 범위

현재 board 요구사항에서 우선 포함하는 범위는 다음과 같다.

- 게시글 목록을 조회할 수 있다.
- 게시글 상세를 조회할 수 있다.
- 게시글을 등록할 수 있다.
- 게시글을 수정할 수 있다.
- 게시글을 삭제할 수 있다.
- 게시글의 기본 정보는 제목과 본문을 포함한다.
- 최소한의 생성/수정 시각 같은 기본 관리 정보가 필요하다는 점을 전제로 한다.

현재는 게시판의 **기본 CRUD 흐름**을 첫 범위로 본다.

---

## 6. 현재 제외 범위

현재 단계에서 직접 포함하지 않는 범위는 다음과 같다.

- 사용자 회원가입 / 로그인
- 권한 관리
- 댓글
- 첨부파일
- 대댓글
- 좋아요 / 추천
- 검색 고도화
- 정렬 / 필터 고도화
- 운영자 기능
- 알림
- 통계
- 배포/운영 기능

이 항목들은 필요성이 확인되면 후속 요구사항 후보로 검토한다.

---

## 7. 기능 요구사항

### 7.1 게시글 목록 조회

현재 사용자는 게시글 목록을 볼 수 있어야 한다.

현재 단계에서 중요한 요구사항은 다음과 같다.

- 게시글 목록이 존재해야 한다.
- 각 항목에서 최소한 제목과 생성/수정 흐름으로 이어질 수 있는 식별 정보가 보여야 한다.
- 상세 조회로 이동할 수 있어야 한다.

### 7.2 게시글 상세 조회

현재 사용자는 게시글 하나의 내용을 볼 수 있어야 한다.

현재 단계에서 중요한 요구사항은 다음과 같다.

- 제목과 본문을 확인할 수 있어야 한다.
- 수정과 삭제 흐름으로 이어질 수 있어야 한다.

### 7.3 게시글 등록

현재 사용자는 새 게시글을 등록할 수 있어야 한다.

현재 단계에서 중요한 요구사항은 다음과 같다.

- 제목은 필수 입력으로 본다.
- 본문은 필수 입력으로 본다.
- 등록이 완료되면 목록 또는 상세 흐름으로 자연스럽게 이어질 수 있어야 한다.

### 7.4 게시글 수정

현재 사용자는 기존 게시글을 수정할 수 있어야 한다.

현재 단계에서 중요한 요구사항은 다음과 같다.

- 기존 제목과 본문을 수정할 수 있어야 한다.
- 수정 후 결과를 확인할 수 있어야 한다.

### 7.5 게시글 삭제

현재 사용자는 기존 게시글을 삭제할 수 있어야 한다.

현재 단계에서 중요한 요구사항은 다음과 같다.

- 삭제 대상이 무엇인지 분명해야 한다.
- 삭제 후 목록 흐름으로 복귀할 수 있어야 한다.

---

## 8. 데이터 요구사항

현재 단계에서 게시글은 최소한 다음 정보를 가진다고 본다.

- 식별자
- 제목
- 본문
- 생성 시각
- 수정 시각

이 문서는 데이터 구조를 상세 설계하지 않는다.  
현재는 **어떤 정보가 필요하다고 보는가** 수준만 고정한다.

---

## 9. 입력 검증 요구사항

현재 단계에서 필요한 최소 입력 검증 요구사항은 다음과 같다.

- 제목은 비어 있으면 안 된다.
- 본문은 비어 있으면 안 된다.
- 잘못된 입력에 대해 사용자가 다시 시도할 수 있어야 한다.

검증 방식, 에러 응답 형식, 예외 처리 구조는  
후속 설계 문서와 standards 적용 문서에서 구체화한다.

---

## 10. 비기능 요구사항

현재 단계에서 최소한 확인해야 하는 비기능 요구사항은 다음과 같다.

- 요구사항과 설계, 구현 계획 문서의 역할이 섞이지 않아야 한다.
- AI 협업과 사람이 함께 검토하기 쉬운 구조를 유지해야 한다.
- 과도한 확장보다 현재 범위의 명확성이 우선이다.

현재는 성능, 보안, 운영, 확장성 목표를 상세 수치로 확정하지 않는다.
