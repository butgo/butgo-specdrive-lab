# 01-overview.md

## 1. 문서 목적

이 문서는 board 프로젝트의 출발 문서다.

현재 단계에서 board 프로젝트의 목적, 검증 방향, 현재 범위, 제외 범위, 기술 방향, 구조 방향을 간단히 고정한다.

이 문서는 상세 요구사항, 상세 설계, 상세 구현 계획을 모두 담는 문서가 아니다.  
상세 내용은 후속 요구사항 문서, 구조 문서, 구현 계획 문서로 분리한다.

---

## 2. 프로젝트 한 줄 정의

board는 specdrive 방식으로 실제 개발을 검증하기 위한 Spring Boot 기반 layered 게시판 프로젝트다.

---

## 3. 프로젝트 배경

specdrive는 개발 스펙 문서를 기반으로 AI 협업을 실행하는 도구이며,  
board는 그 방식이 실제 애플리케이션 프로젝트에도 유효한지 검증하기 위한 첫 번째 실제 프로젝트다.

board는 specdrive의 하위 기능이 아니라,  
specdrive가 다루는 실제 애플리케이션 프로젝트로 본다.

이 프로젝트의 목적은 게시판 자체를 빠르게 구현하는 것만이 아니다.  
문서 우선의 AI 협업 흐름이 실제 프로젝트에서도 작동하는지 확인하고,  
요구사항 → 설계 → 구현 계획 → 상태 → 히스토리 흐름을 실제로 운영해 보는 데 의미가 있다.

---

## 4. 현재 검증 목표

현재 board 프로젝트의 검증 목표는 다음과 같다.

- Spring Boot 기반 layered 구조로 최소 게시판 애플리케이션을 문서화하고 구현할 수 있는지 검증한다.
- specdrive의 문서 기반 `doc` 흐름을 실제 프로젝트 문서에 적용할 수 있는지 검증한다.
- 현재 최소 게시판 범위를 기준으로 후속 확장 여지와 구조 경계를 무리 없이 남길 수 있는지 검증한다.

현재 단계에서 board는 완성된 제품보다  
**문서 기반 개발 흐름 검증 대상 프로젝트**로 보는 것이 더 중요하다.

---

## 5. 현재 범위

현재 단계에서 우선 범위는 다음과 같다.

- 게시판 기본 구조
- 게시글 작성, 조회, 수정, 삭제의 최소 흐름
- 문서 우선 개발 흐름 검증에 필요한 최소 문서 세트 작성

현재는 초기 게시판 프로젝트 기준의 최소 구현 범위를 우선하며,  
실제 요구사항과 설계는 후속 문서에서 구체화한다.

---

## 6. 제외 범위

현재 단계에서는 다음을 우선 범위에 포함하지 않는다.

- 헥사고날 구조를 전제로 한 문서 체계 확정
- auth와 board의 독립 배포 구조 확정
- 멀티테넌시
- 대규모 검색, 통계, 분석 기능
- 과도한 공통 모듈 일반화
- 공통 모듈의 완성형 제품화
- board 상세 API, DB, 패키지 설계 본문 확정
- 댓글 구조 확정
- 사용자 인증 연계 방식 확정
- 관리자 기능 범위 확정
- 모바일 앱 구조 착수

현재 단계에서는 미래 확장을 이유로 현재 구조를 과도하게 복잡하게 만들지 않는다.

---

## 7. 기술 방향

현재 board 프로젝트의 기술 방향은 다음과 같다.

- Spring Boot 기반 애플리케이션
- layered architecture 우선 적용
- JPA 기반 persistence 우선 검토
- DB 마이그레이션 도구는 Flyway 사용 가능성을 고려
- API 중심 웹 애플리케이션 구조 우선

현재는 공통 standards 문서를 참조하되,  
board에 실제로 어떻게 적용할지는 board 문서에서 명확히 적는다.

---

## 8. 구조 방향

현재 board 프로젝트는 Spring Boot 기반 layered 구조를 우선 검증 대상으로 삼는다.

공통 로깅 기능은 다른 프로젝트에서도 재사용할 수 있도록  
starter 형태의 분리 가능성을 고려한다.

인증(auth)과 게시판(board)은 현재 단계에서는  
하나의 애플리케이션 안에서 함께 동작하는 구조를 우선 검토한다.

다만 장기적으로는 auth와 board를 분리 가능한 서비스 모듈 구조로 확장할 수 있도록 경계를 의식한다.

현재 단계에서는 실제 구현과 검증에 필요한 최소 구조를 우선하며,  
과도한 모듈 분리나 독립 배포 구조는 먼저 확정하지 않는다.  
필요가 검증되면 이후 구조 문서에서 구체화한다.

---

## 9. 문서 작성 방향

board 문서는 specdrive 운영 규칙 설명 문서가 아니라,  
실제 애플리케이션 개발 내용을 다루는 문서여야 한다.

따라서 board 문서에서는 다음을 우선한다.

- 현재 채택한 범위와 방향을 먼저 기록한다.
- 요구사항, 설계, 구현 계획, 상태, 히스토리를 가능한 한 분리한다.
- overview 문서에는 상세 설계를 과도하게 넣지 않는다.
- overview 문서에서 상세 API, DB, 패키지 설계 본문을 확정하지 않는다.
- 현재 범위와 후속 후보를 구분한다.
- AI가 읽기 쉬운 구조를 유지한다.

이후 세부 내용은 후속 문서로 분리한다.

## 11. 한 줄 정리

board는 Spring Boot 기반 layered 게시판 애플리케이션을 실제로 만들면서,  
specdrive의 문서 기반 AI 협업 흐름이 실제 프로젝트에서도 유효한지 검증하기 위한 첫 번째 실제 프로젝트다.
